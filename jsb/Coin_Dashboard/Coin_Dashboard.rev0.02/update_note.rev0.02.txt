Clone_Table.py
이전 코드는 단순히 원본 테이블을 복제하는 기능만 있었지만, 수정된 코드는 복제 후 원본 테이블의 PRIMARY KEY를 찾아 새로 만들어진 테이블에 동일한 PRIMARY KEY를 추가하는 기능이 추가되었습니다. 즉, 바뀐 점은 원본 테이블의 PRIMARY KEY 정보를 조회한 뒤, 복제된 테이블에 같은 PRIMARY KEY를 설정하는 로직이 추가된 것입니다.

Indicator.py
import 방식 단순화: 여러 파일에서 함수를 개별 import하던 것을 하나의 패키지에서 필요한 함수만 import하도록 변경하여 관리 및 유지보수가 쉬워졌습니다.
함수 파라미터 변경: DB 커넥션과 테이블명을 함수 외부에서 주입받도록 하여 테스트와 재사용성을 높였습니다.
쿼리 정렬 변경: 최근 날짜 기준으로 내림차순 정렬해 실시간성 분석에 유리해졌습니다.
데이터 검증 로직 추가: 비어있는 데이터프레임이나 거래량 0 이상 조건을 만족하지 않으면 계산을 건너뛰어 오류와 불필요한 계산을 방지합니다.
지표 계산 기간 변경: EMA, 볼린저밴드, STO 등의 기간을 조정해 데이터 특성과 성능을 개선했습니다.
NaN 처리 강화: 조건부 NaN 처리로 데이터 유효성에 따라 처리 로직을 더 견고하게 했습니다.
DB 업데이트 효율화: 개별 행 업데이트 대신 executemany()를 사용해 대량 업데이트 성능이 개선되었습니다.
커넥션 관리 개선: 함수 내부에서 커넥션을 닫지 않아 외부에서 커넥션 관리를 할 수 있게 했습니다.
